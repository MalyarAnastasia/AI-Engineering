## 1. Dataset

- **Какой датасет выбран:** `S06-hw-dataset-01.csv`

- **Размер:** 12000 строк, 30 столбцов

- **Целевая переменная:** `target` (бинарная классификация)
  - Класс 0: 8119 примеров (67.7%)
  - Класс 1: 3881 пример (32.3%)
  
  Классов немного несбалансировано - класс 0 встречается в два раза чаще.

- **Признаки:** Всего 28 признаков (без id и target)
  - Числовые признаки: 24 штуки (num01 - num24), это обычные числа
  - Категориальные признаки: 3 штуки (cat_contract, cat_region, cat_payment), это целые числа, которые означают категории
  - Признак времени: tenure_months - сколько месяцев

Пропусков нет, данные синтетические.

## 2. Protocol

- **Разбиение:** train/test split
  - Test: 20% (2400 примеров)
  - Train: 80% (9600 примеров)
  - `random_state=42` - чтобы результаты были одинаковые при повторном запуске
  - Стратификация (`stratify=y`) - чтобы в train и test было одинаковое соотношение классов

- **Подбор:** Кросс-валидация на train
  - 5 фолдов (cv=5)
  - Оптимизировали ROC-AUC
  - Использовали GridSearchCV
  - Подбирали параметры для DecisionTreeClassifier, RandomForestClassifier и GradientBoostingClassifier

- **Метрики:** accuracy, F1, ROC-AUC
  - **Accuracy** - просто доля правильных ответов. Но при дисбалансе классов может быть обманчивой
  - **F1** - учитывает и точность, и полноту. Полезно при дисбалансе
  - **ROC-AUC** - показывает, насколько хорошо модель различает классы. Хорошо для бинарной классификации

Эти метрики нужны, потому что у нас дисбаланс классов (67% против 33%), и нужно смотреть не только на accuracy.

## 3. Models

Использовала следующие модели:

- **DummyClassifier** - самая простая модель, всегда предсказывает класс 0 (самый частый). Это baseline, чтобы понять минимальный уровень.

- **LogisticRegression** - базовая модель из прошлых семинаров. С нормализацией признаков. 

- **DecisionTreeClassifier** - дерево решений
  - Подбирала `max_depth` (глубина дерева): [3, 5, 7, 10, 15, None]
  - Подбирала `min_samples_leaf` (минимум примеров в листе): [1, 2, 5, 10]
  - Всего 24 комбинации
  - Лучшие параметры: max_depth=10, min_samples_leaf=10

- **RandomForestClassifier** - случайный лес (много деревьев)
  - Подбирала n_estimators (количество деревьев): [50, 100]
  - Подбирала max_depth: [5, 10, 15, None]
  - Подбирала min_samples_leaf: [1, 2, 5]
  - Подбирала max_features: ['sqrt', 'log2']
  - Всего 48 комбинаций
  - Лучшие параметры: n_estimators=100, max_depth=None, min_samples_leaf=2, max_features='sqrt'

- **GradientBoostingClassifier** - градиентный бустинг (деревья обучаются последовательно)
  - Подбирала n_estimators: [50, 100]
  - Подбирала max_depth: [3, 5]
  - Подбирала learning_rate: [0.1, 0.2]
  - Подбирала min_samples_leaf: [1, 2]
  - Всего 16 комбинаций (упростила сетку, чтобы быстрее работало)
  - Лучшие параметры: n_estimators=100, max_depth=7, learning_rate=0.2, min_samples_leaf=5

Все модели обучались на train, а на test проверяли только один раз в конце.

## 6. Conclusion

Что я поняла:

1. Дерево решений легко переобучается. Нужно ограничивать глубину (max_depth) и количество примеров в листе (min_samples_leaf), иначе дерево запомнит тренировочные данные, но плохо будет работать на новых.

2. Ансамбли работают лучше одиночных моделей. RandomForest и GradientBoosting дали ROC-AUC 0.96-0.97, а одиночное дерево только 0.91. Когда объединяешь много слабых моделей, получается сильная.

3. Подбор параметров долгий. GridSearchCV с большими сетками может работать долго. Пришлось упростить сетку для GradientBoosting.

4. Честный ML-протокол важен. Нужно разделить данные на train/test, подбирать параметры только на train, а test использовать один раз в конце. Если смотреть на test много раз или подбирать на нем параметры, оценки будут завышенными.

5. При дисбалансе классов одной accuracy мало. Нужно смотреть на F1 и ROC-AUC тоже. ROC-AUC особенно хорош для бинарной классификации.

6. Permutation importance помогает понять, что важно. Оказалось, что только несколько признаков (num19, num18, num07) действительно важны. Это можно использовать, чтобы упростить модель, убрав неважные признаки.
