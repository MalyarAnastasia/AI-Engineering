# какие команды CLI доступны

- overview
Размеры датасета (строки/столбцы) и таблица с информацией по каждой колонке (тип, пропуски, уникальные значения и т.д.).

- report

Основной отчет с эвристиками качества, таблицы в CSV, графики в PNG

## Новые параметры команды report

- title
Изменяет главный заголовок отчёта в файле report.md. Позволяет присвоить отчёту осмысленное имя.
- top-k-categories
Контролирует глубину анализа категориальных признаков. Определяет, сколько самых частых значений (Top-K) будет: 1. Рассчитано в core.py. 2. Сохранено в отдельных CSV-файлах (в папке top_categories/). 3. Упомянуто в заголовке раздела "Категориальные признаки" в report.md
- min-missing-share
Устанавливает порог проблемных пропусков (доля от 0.0 до 1.0). В отчёте report.md создаётся новый подраздел, который явно перечисляет все колонки, чья доля пропусков превышает или равна этому порогу.

## пример вызова `eda-cli report` с новыми опциями.

uv run eda-cli report data/example.csv --out-dir reports_test --title "Тестовый отчет с новыми параметрами" --top-k-categories 10

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.
