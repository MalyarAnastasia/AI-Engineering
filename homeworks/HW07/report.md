## 2. Protocol

Опишу свой протокол работы с данными:

- Препроцессинг: для всех датасетов использовала StandardScaler - это стандартизация, которая приводит все признаки к одному масштабу (среднее = 0, стандартное отклонение = 1). Это очень важно, потому что без этого признаки в разных шкалах будут влиять на результат по-разному. Пропусков не было, поэтому imputation не нужен был. Категориальных признаков тоже не было, поэтому encoding не требовался. PCA использовала только для визуализации в 2D, не для препроцессинга.

- Поиск гиперпараметров:
  - Для KMeans: перебирала k от 2 до 20. Для каждого k запускала алгоритм с random_state=42 и n_init=10 (это означает, что алгоритм запускается 10 раз и выбирается лучший результат).
  - Для DBSCAN: перебирала eps от 0.1 до 2.0 (с шагом 0.1) для первого датасета и от 0.1 до 1.5 для остальных, и min_samples = [3, 5, 7]. Пробовала разные комбинации и выбирала ту, где silhouette score максимальный, но при этом было не слишком много шума (не больше 50% точек).
  - Выбор "лучшего": сравнивала методы по silhouette score - чем он выше, тем лучше. Также смотрела на Davies-Bouldin (чем ниже, тем лучше) и Calinski-Harabasz (чем выше, тем лучше), но основной критерий был silhouette.

- Метрики: использовала три метрики:
  - Silhouette Score: показывает, насколько хорошо точки разделены на кластеры. Диапазон от -1 до 1, чем выше, тем лучше.
  - Davies-Bouldin Score: показывает, насколько компактны и разделены кластеры. Чем ниже, тем лучше.
  - Calinski-Harabasz Score: отношение межкластерной дисперсии к внутрикластерной. Чем выше, тем лучше.
  - Для DBSCAN: когда были точки с меткой -1 (шум), я считала метрики только для нешумовых точек, потому что для шума метрики не имеют смысла. Также фиксировала долю шума.

- Визуализация: использовала PCA для снижения размерности до 2D и визуализации результатов. PCA объясняет часть дисперсии данных (обычно 60-80% для первых двух компонент). t-SNE не использовала, потому что это сложнее интерпретировать и требует больше времени.

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

KMeans "ломается" в следующих случаях:
- Когда кластеры не круглые (dataset 2) - KMeans ищет сферические кластеры, поэтому не может правильно разделить данные сложной формы
- Когда кластеры разной плотности (dataset 3) - KMeans предполагает, что все кластеры примерно одинаковой плотности, поэтому может неправильно разделить данные

DBSCAN выигрывает когда:
- Есть нелинейная структура (dataset 2) - может находить кластеры произвольной формы
- Есть выбросы (dataset 2, 3) - автоматически выделяет их как шум
- Кластеры разной плотности (dataset 3) - при правильном подборе параметров может работать с разной плотностью

Что сильнее всего влияло на результат:
- Масштабирование - это было критически важно! Без него результаты были бы неправильными, потому что признаки в разных шкалах доминировали бы друг над другом
- Структура данных - для круглых кластеров KMeans работает хорошо, для сложных форм нужен DBSCAN
- Выбросы - DBSCAN их автоматически находит, а KMeans пытается включить в кластеры
- Плотность кластеров - разная плотность создает проблемы для обоих методов, но DBSCAN справляется лучше при правильных параметрах

### 5.2 Устойчивость (обязательно для одного датасета)

Проверку устойчивости делала для dataset 3 с KMeans:
- Запускала KMeans 5 раз с разными random_state: [42, 123, 456, 789, 999]
- Сравнивала результаты с базовым запуском (random_state=42) используя Adjusted Rand Index (ARI)
- ARI показывает, насколько похожи два разбиения данных на кластеры (от 0 до 1, где 1 - полностью одинаковые)

Результаты: все 5 запусков дали ARI = 1.0, что означает, что результаты полностью идентичны базовому. Это говорит о том, что KMeans на этом датасете очень устойчив - при разных начальных условиях получается одно и то же разбиение. Это хорошо, потому что означает, что результат не зависит от случайности.

Вывод: KMeans на dataset 3 очень устойчив. Это может быть потому, что кластеры хорошо разделены и алгоритм всегда находит одно и то же оптимальное решение независимо от начальной инициализации.

### 5.3 Интерпретация кластеров

Интерпретировала кластеры визуально через PCA-визуализацию:
- Смотрела на графики, где точки раскрашены по кластерам
- Видела, как кластеры расположены в пространстве первых двух главных компонент
- Оценивала, насколько хорошо кластеры разделены визуально

Выводы:
- На dataset 1 кластеры хорошо разделены и компактны - это подтверждает хорошие метрики
- На dataset 2 видно, что DBSCAN нашла кластеры сложной формы, которые KMeans не смог бы правильно разделить
- На dataset 3 видно разную плотность кластеров, что объясняет, почему DBSCAN показал лучший результат

К сожалению, не делала детального анализа средних значений признаков по кластерам, но визуальная оценка через PCA дала хорошее понимание структуры данных.

## 6. Conclusion

Чему научилась:

1. Масштабирование критически важно для distance-based методов - без него результаты бессмысленны, потому что признаки в разных шкалах влияют на результат неправильно.

2. Выбор алгоритма зависит от структуры данных - KMeans хорош для круглых кластеров одинакового размера, а DBSCAN лучше для сложных форм и выбросов.

3. Подбор параметров - это сложно, особенно для DBSCAN. Нужно пробовать много комбинаций и смотреть не только на метрики, но и на долю шума и количество кластеров.

4. Метрики качества помогают сравнивать методы, но их нужно правильно интерпретировать - для DBSCAN нужно считать метрики только для нешумовых точек.

5. Визуализация через PCA очень помогает понять, что происходит с данными и насколько хорошо работает кластеризация.

6. Устойчивость важна - хорошо, когда алгоритм дает похожие результаты при разных запусках, это означает надежность.

7. Нет универсального лучшего алгоритма - каждый подходит для своих задач, и нужно пробовать разные методы.

8. Unsupervised-эксперимент требует аккуратности - нужно правильно препроцессить данные, подобрать параметры, оценить результаты и интерпретировать их.
