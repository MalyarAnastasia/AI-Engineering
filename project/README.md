## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - `01_eda.ipynb` – разведочный анализ данных
  - `02_clustering.ipynb` – кластеризация и сегментация рынка
  - `03_baselines.ipynb` – baseline модели (Linear Regression, Decision Tree)
  - `04_advanced_models.ipynb` – продвинутые модели (CatBoost, Bagging)
  - `05_model_selection.ipynb` – сравнение моделей и выбор финальной
- `src/` – основной код проекта:
  - `src/models/` – обучение и применение моделей (train.py)
  - `src/service/` – FastAPI сервис (app.py)
- `data/` – данные проекта:
  - `all_v2.csv` – полный датасет (4.5M+ записей)
  - `cleaned_data.csv` – очищенный датасет (используется для обучения)
- `configs/` – конфигурационные файлы:
  - `config.yaml` – общие настройки
  - `.env.example` – шаблон переменных окружения
- `tests/` – тесты (юнит-тесты, smoke-тесты).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.11`
- Docker (опционально, для контейнеризации)
- 4GB+ RAM для обучения моделей

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt

# Создать .env файл из шаблона
cp configs/.env.example .env
```

## 4. Как запустить проект

### 4.1. Обучение модели

```bash
cd project
source .venv/bin/activate      # Windows: .venv\Scripts\activate
python -m src.models.train
```

Модель сохранится в `artifacts/model.joblib`.

### 4.2. Запуск API сервиса

```bash
cd project
source .venv/bin/activate      # Windows: .venv\Scripts\activate
python -m src.service.app
```

Сервис запустится на порту `8000` (настраивается в `.env`).

Доступные эндпоинты:
- `GET /health` – проверка работоспособности сервиса
- `POST /predict` – оценка стоимости недвижимости

Пример запроса:

```bash
curl -X POST "http://localhost:8000/predict" \
  -H "Content-Type: application/json" \
  -d '{
    "area": 65.5,
    "kitchen_area": 10.0,
    "rooms": 2,
    "geo_lat": 55.7558,
    "geo_lon": 37.6173
  }'
```

### 4.3. Запуск через Docker

```bash
cd project
docker build -t real-estate-api .
docker run -p 8000:8000 --env-file configs/.env real-estate-api
```

## 5. Данные

Используется реальный датасет объявлений о продаже недвижимости в Москве, собранный с различных сайтов недвижимости и доступный на Kaggle:

- **Источник:** Kaggle - Real Estate Dataset (собран с разных сайтов недвижимости)
- **Файлы:**
  - `data/all_v2.csv` – полный датасет (4.5M+ записей)
  - `data/cleaned_data.csv` – очищенный и отфильтрованный датасет (используется для обучения)
- **Обработка данных:**
  - Удаление пропусков и выбросов (IQR метод)
  - Фильтрация по положительным значениям (area > 0, kitchen_area > 0, price > 0, rooms >= 0)
  - Очистка данных выполняется в `src/models/train.py`

**Основные признаки:**
- `area` – общая площадь (м²)
- `kitchen_area` – площадь кухни (м²)
- `rooms` – количество комнат
- `geo_lat`, `geo_lon` – координаты
- `price` – цена (целевая переменная)
- `region` – регион (код)

**Предобработка:**
- Очистка от выбросов (IQR метод)
- Фильтрация по положительным значениям
- Удаление пропусков

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**
- Используется только базовый набор признаков (без расстояний до метро, инфраструктуры)
- Модель обучена на данных за определённый период (может устаревать)
- Нет системы переобучения модели на новых данных
- Рекомендации основаны только на географическом proximity

**Дальнейшее развитие:**
- Добавление внешних признаков (расстояние до метро, парков, школ)
- Реализация RAG для анализа описаний объектов
- Добавление системы A/B тестирования для моделей
- Интеграция с картами для визуализации
- Добавление системы мониторинга качества модели в продакшене
