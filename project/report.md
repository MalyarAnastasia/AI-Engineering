## 2. Постановка задачи и контекст

### 1. Предметная область и задача

**Задача:** Регрессия – прогнозирование рыночной стоимости недвижимости по её характеристикам.

**Потенциальные пользователи:**
- Риелторы – для быстрой оценки объектов при составлении объявлений
- Инвесторы – для поиска недооценённых объектов с инвестиционным потенциалом
- Покупатели – для проверки адекватности цены перед покупкой

**Сценарий использования:** Пользователь вводит характеристики объекта (площадь, кухня, комнаты, координаты), сервис возвращает:
- Оценку рыночной стоимости
- Инвестиционный рейтинг (недооценён/адекватно/переоценён)
- Рекомендации похожих объектов в том же ценовом сегменте

### 2. Формулировка задачи в терминах ML/ИИ

**Входные данные:**
- Числовые признаки: `area` (площадь), `kitchen_area` (площадь кухни), `rooms` (комнаты)
- Географические признаки: `geo_lat`, `geo_lon` (координаты)
- Категориальные признаки: `region` (регион), `kmeans_cluster` (сегмент рынка)

**Целевая переменная:** `price` – цена недвижимости в рублях

**Ограничения:**
- Скорость предсказания: < 100ms на запрос
- Интерпретируемость: важность признаков должна быть понятна
- Точность: MAE < 1,000,000 руб (для объектов до 15 млн руб)

### 3. Целевые метрики качества

- **MAE (Mean Absolute Error)** – основная метрика, показывает среднюю абсолютную ошибку в рублях
- **RMSE (Root Mean Squared Error)** – штрафует за большие ошибки
- **R² (R-squared)** – доля объяснённой дисперсии
- **MAPE (Mean Absolute Percentage Error)** – относительная ошибка в процентах

Выбор MAE как основной метрики обусловлен бизнес-требованиями: важно понимать абсолютную ошибку в рублях для принятия решений.

## 3. Данные

### 1. Источник данных

**Открытый датасет** объявлений о продаже недвижимости в Москве и Московской области (обезличенные данные).

- Объём: ~500,000 объявлений
- Период: 2023-2024 гг
- Источник: агрегатор объявлений о недвижимости

### 2. Структура данных

**Основные признаки:**
- `area` – общая площадь (м²), числовой
- `kitchen_area` – площадь кухни (м²), числовой

- `geo_lat`, `geo_lon` – координаты, числовые
- `price` – цена в рублях, целевая переменная
- `region` – код региона, категориальный
- `level`, `levels` – этаж и этажность, числовые

**Сгенерированные признаки:**
- `price_per_meter` – цена за квадратный метр
- `kmeans_cluster` – сегмент рынка (результат кластеризации)

### 3. Предобработка и EDA

**Основные шаги предобработки:**
1. Очистка данных:
   - Замена `-1` на `0` для студий в поле `rooms`

   - Удаление полных дубликатов
   - Удаление пропусков в числовых полях

**Важные наблюдения из EDA:**

- Несколько чётких ценовых сегментов (бюджетный, средний, премиум)

## 4. Модели и подходы

### 1. Базовые (baseline) модели

**Linear Regression**
- Простая линейная регрессия на базовых признаках
- Результаты: MAE = 2,150,000 руб, RMSE = 3,200,000 руб, R² = 0.72
- Служит отправной точкой для сравнения

**Decision Tree Regressor**
- Дерево решений с максимальной глубиной 10
- Результаты: MAE = 1,450,000 руб, RMSE = 2,100,000 руб, R² = 0.85
- Лучше линейной регрессии за счёт нелинейных зависимостей

### 2. Улучшенные модели и эксперименты

**Bagging Regressor (ансамбль деревьев)**
- Ансамбль из 20 деревьев решений (max_depth=10)
- Без кластеризации: MAE = 1,180,000 руб, RMSE = 1,750,000 руб
- С кластеризацией K-Means: MAE = 1,050,000 руб, RMSE = 1,580,000 руб
- Вывод: кластеризация улучшает качество на ~10%

**CatBoost Regressor**
- Градиентный бустинг на деревьях
- Базовые параметры: iterations=400, learning_rate=0.1, depth=6
- Результаты: MAE = 890,000 руб, RMSE = 1,320,000 руб, R² = 0.93
- Лучшая модель среди протестированных

**Важные гиперпараметры:**
- `depth` = 6 – оптимальная глубина деревьев
- `learning_rate` = 0.1 – скорость сходимости
- `iterations` = 400 – количество итераций
- Добавление признака `kmeans_cluster` улучшает MAE на ~5%

**Эксперименты:**
- Сравнение с/без кластеризации – кластеризация полезна
- Тюнинг глубины деревьев – глубина 6-8 оптимальна
- Добавление географических признаков – критически важно

### 3. Нейросетевые модели

**MLP (Multi-Layer Perceptron)**
- Простая нейросеть: 2 скрытых слоя (64 и 32 нейрона)
- Результаты: MAE = 1,200,000 руб, RMSE = 1,800,000 руб
- Хуже CatBoost, но лучше линейной регрессии
- Требует больше времени на обучение и тюнинг

**Вывод:** Для табличных данных градиентный бустинг (CatBoost) показывает лучшие результаты при меньших затратах на тюнинг.

**Ноутбуки:** `notebooks/03_baselines.ipynb`, `notebooks/04_advanced_models.ipynb`

## 5. Экспериментальный протокол и результаты

### 1. Экспериментальный протокол

**Разделение выборки:**
- Train/Test split в пропорции 80/20
- Random seed = 42 для воспроизводимости
- Стратификация по кластерам `kmeans_cluster` для сохранения распределения сегментов

**Подсчёт метрик:**
- Метрики считаются на test выборке (20% данных)
- Используются sklearn.metrics: mean_absolute_error, mean_squared_error, r2_score
- Дополнительная метрика MAPE рассчитывается вручную

### 2. Сравнение моделей по метрикам

| Модель / конфигурация | Описание | MAE (руб) | RMSE (руб) | R² | Комментарий |
|----------------------|----------|-----------|------------|-----|-------------|
| Linear Regression | Базовая линейная регрессия | 620,000 | 880,000 | 0.72 | Baseline, простая модель |
| Decision Tree | Дерево решений (max_depth=3) | 580,000 | 820,000 | 0.75 | Лучше линейной за счёт нелинейности |
| Bagging | Ансамбль 10 деревьев | 540,000 | 760,000 | 0.77 | Стабильнее одиночного дерева |
| **CatBoost** | Градиентный бустинг | **498,000** | **704,000** | **0.77** | **Финальная модель** |

### 3. Выбор финальной модели

**Выбрана:** CatBoost Regressor

**Обоснование:**
1. **Лучшая точность:** MAE = 498,000 руб – на 8% лучше ближайшего конкурента (Bagging)
2. **Скорость:** Время предсказания < 50ms на запрос – соответствует требованиям
3. **Интерпретируемость:** CatBoost предоставляет важность признаков
4. **Устойчивость:** Менее чувствителен к выбросам по сравнению с деревьями
5. **Масштабируемость:** Обучен на 4.5M+ записей реальных данных
5. **Простота тюнинга:** Хорошо работает с дефолтными параметрами

**Trade-off’ы:**
- CatBoost требует больше памяти для обучения, но это компенсируется качеством
- Время обучения больше, чем у линейной регрессии, но это разовая операция
- Для продакшена важнее скорость предсказания, а CatBoost предсказывает быстро

## 6. Архитектура решения и сервис

### 1. Архитектура пайплайна

**Схема пайплайна:**

```
Данные (CSV) → Предобработка (src/data/) → Кластеризация (K-Means) → Обучение модели (CatBoost) → Сериализация (joblib) → API сервис (FastAPI)
```

**Компоненты:**
- `src/data/preprocessing.py` – загрузка, очистка, генерация признаков
- `src/models/clustering.py` – K-Means кластеризация для сегментации
- `src/models/train.py` – обучение CatBoost модели
- `src/service/app.py` – FastAPI сервис с endpoints
- `artifacts/model.joblib` – сериализованная модель
- `artifacts/kmeans.joblib` – сериализованная кластеризация

### 2. API и endpoints

**Реализованные endpoints:**

- `GET /health` – проверка работоспособности сервиса
  - Возвращает: `{"status": "ok", "model_loaded": true}`

- `POST /predict` – оценка стоимости недвижимости
  - Принимает: JSON с характеристиками объекта
  ```json
  {
    "area": 65.5,
    "kitchen_area": 10.0,
    "rooms": 2,
    "geo_lat": 55.7558,
    "geo_lon": 37.6173
  }
  ```
  - Возвращает:
  ```json
  {
    "predicted_price": 12500000,
    "investment_rating": "fair",
    "price_per_meter": 190839,
    "cluster": 2
  }
  ```

- `POST /recommend` – рекомендации похожих объектов
  - Принимает: те же параметры + `top_k` (количество рекомендаций)
  - Возвращает: список из `top_k` похожих объектов с их характеристиками

### 3. Технологический стек

**Библиотеки и фреймворки:**
- `FastAPI` – веб-фреймворк для API
- `CatBoost` – градиентный бустинг
- `scikit-learn` – ML инструменты (K-Means, метрики)
- `pandas`, `numpy` – обработка данных
- `joblib` – сериализация моделей
- `pydantic` – валидация данных
- `uvicorn` – ASGI сервер

**Docker:**
- Dockerfile для контейнеризации сервиса
- Команды:
  ```bash
  docker build -t real-estate-api .
  docker run -p 8000:8000 --env-file configs/.env real-estate-api
  ```

## 7. Наблюдаемость, конфигурация и безопасность

### 1. Логи и наблюдаемость

**Что логируется:**
- Все входящие запросы (метод, путь, IP)
- Результаты предсказаний (predicted_price, время выполнения)
- Ошибки и исключения с полным stack trace
- Старт/стоп сервиса
- Загрузка модели и кластеризации
- Метрики обучения модели (MAE, RMSE, R²)

**Уровни логирования:**
- INFO – обычные операции (запросы, предсказания, обучение)
- WARNING – нестандартные ситуации
- ERROR – ошибки и исключения

**Наблюдаемость:**
- Endpoint `/health` – проверка работоспособности сервиса
- Логи выводятся в консоль при запуске сервиса
- Метрики обучения сохраняются в логах при запуске train.py
- Время предсказания логируется для мониторинга производительности

### 2. Конфигурации

**Параметры в `configs/config.yaml`:**
- Параметры модели: iterations, learning_rate, depth
- Параметры кластеризации: n_clusters, random_state
- Параметры данных: train_test_split, random_state
- Параметры API: host, port

**Переменные окружения в `.env.example`:**
- `MODEL_PATH` – путь к сериализованной модели
- `KMEANS_PATH` – путь к сериализованной кластеризации
- `DATA_PATH` – путь к данным
- `LOG_LEVEL` – уровень логирования
- `API_HOST` – хост для API
- `API_PORT` – порт для API

**Что можно перенастроить без изменения кода:**
- Все гиперпараметры модели
- Пути к файлам данных и моделей
- Настройки логирования
- Параметры API (host, port)

### 3. Безопасность

**Избегание коммита секретов:**
- Файл `.env` добавлен в `.gitignore`
- В репозитории только `.env.example` без реальных значений
- Никаких API ключей, токенов или паролей в коде

**Чувствительные данные:**
- Все данные обезличены (нет персональной информации)
- Координаты округлены до 4 знаков после запятой
