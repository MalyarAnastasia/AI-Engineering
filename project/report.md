## 2. Постановка задачи и контекст

### 1. Предметная область и задача

**Задача:** Регрессия – прогнозирование рыночной стоимости недвижимости по её характеристикам.

**Потенциальные пользователи:**
- Риелторы – для быстрой оценки объектов при составлении объявлений
- Инвесторы – для поиска недооценённых объектов с инвестиционным потенциалом
- Покупатели – для проверки адекватности цены перед покупкой

**Сценарий использования:** Пользователь вводит характеристики объекта (площадь, кухня, комнаты, координаты), сервис возвращает:
- Оценку рыночной стоимости
- Инвестиционный рейтинг (недооценён/адекватно/переоценён)
- Цену за квадратный метр

### 2. Формулировка задачи в терминах ML/ИИ

**Входные данные:**
- Числовые признаки: `area` (площадь), `kitchen_area` (площадь кухни), `rooms` (комнаты)
- Географические признаки: `geo_lat`, `geo_lon` (координаты)
- Сгенерированный признак: `kmeans_cluster` (сегмент рынка, K-Means)

На этапе предобработки используется поле `region` для фильтрации объектов Москвы и Московской области; в финальную регрессионную модель оно не входит.

**Целевая переменная:** `price` – цена недвижимости в рублях

**Ограничения:**
- Скорость предсказания: < 100ms на запрос
- Интерпретируемость: важность признаков должна быть понятна
- Точность: MAE < 1,000,000 руб (для объектов до 15 млн руб)

### 3. Целевые метрики качества

- **MAE (Mean Absolute Error)** – основная метрика, показывает среднюю абсолютную ошибку в рублях
- **RMSE (Root Mean Squared Error)** – штрафует за большие ошибки
- **R² (R-squared)** – доля объяснённой дисперсии
- **MAPE (Mean Absolute Percentage Error)** – относительная ошибка в процентах

Выбор MAE как основной метрики обусловлен бизнес-требованиями: важно понимать абсолютную ошибку в рублях для принятия решений.

## 3. Данные

### 1. Источник данных

**Реальный датасет** объявлений о продаже недвижимости, собранный с различных сайтов и доступный на Kaggle.

- Исходный объём: ~5,5 млн записей (`data/all_v2.csv`)
- После очистки, IQR и фильтра по Москве и МО: **641 362** записи (`data/cleaned_data.csv`)
- Источник: Kaggle — Real Estate Dataset

### 2. Структура данных

**Основные признаки:**
- `area` – общая площадь (м²), числовой
- `kitchen_area` – площадь кухни (м²), числовой
- `rooms` – количество комнат, числовой (0 = студия)
- `geo_lat`, `geo_lon` – координаты, числовые
- `price` – цена в рублях, целевая переменная

**Сгенерированные признаки:**
- `kmeans_cluster` – сегмент рынка (результат кластеризации)

### 3. Предобработка и EDA

**Основные шаги предобработки** (`src/data/preprocessing.py`, `src/data/prepare.py`):
1. Очистка данных:
   - Замена студий (`rooms = -1` → `0`), удаление дубликатов и отрицательных значений
   - Удаление пропусков в числовых полях
   - Нормализация кодов `region` и **фильтрация по Москве и МО**
   - Удаление выбросов методом IQR для `price`, `area`, `kitchen_area`

2. Кластеризация:
   - K-Means по признакам `price`, `area`, `kitchen_area`, `geo_lat`, `geo_lon`
   - Оптимальное количество кластеров: 4 (по методу локтя)
   - Результат сохраняется как признак `kmeans_cluster`

**Важные наблюдения из EDA:**
- Сильная корреляция между `area` и `price` (0.85)
- Выраженная географическая сегментация: центральные объекты дороже при прочих равных
- Несколько чётких ценовых сегментов (бюджетный, средний, премиум)

**Ноутбуки:** `notebooks/01_eda.ipynb`, `notebooks/02_clustering.ipynb`

## 4. Модели и подходы

### 1. Базовые (baseline) модели

**Linear Regression** (ноутбук `03_baselines`, без `kmeans_cluster`)
- Простая линейная регрессия на базовых признаках
- Результаты на выборке 100K: MAE ≈ 1 458 000 руб, R² ≈ 0.25
- Служит отправной точкой для сравнения

**Decision Tree Regressor** (ноутбук `03_baselines`, max_depth=3)
- Результаты на выборке 100K: MAE ≈ 1 312 000 руб, R² ≈ 0.38
- Лучше линейной регрессии за счёт нелинейных зависимостей

### 2. Улучшенные модели и эксперименты

**Bagging Regressor** (ноутбук `04_advanced_models`, с `kmeans_cluster`)
- Ансамбль из 10 деревьев решений (max_depth=3)
- Результаты на выборке 100K: MAE ≈ 839 000 руб, R² ≈ 0.71

**CatBoost Regressor** (ноутбук `04_advanced_models` и `src/models/train.py`)
- Градиентный бустинг на деревьях
- Параметры: iterations=400, learning_rate=0.1, depth=6
- На выборке 100K (с кластером): MAE ≈ 398 000 руб, R² ≈ 0.94
- **Финальная модель** на полном датасете (641K): MAE ≈ **422 000** руб, RMSE ≈ **508 000** руб, R² ≈ **0.94**

**Важные гиперпараметры:**
- `depth` = 6 – оптимальная глубина деревьев
- `learning_rate` = 0.1 – скорость сходимости
- `iterations` = 400 – количество итераций
- Добавление признака `kmeans_cluster` улучшает MAE на ~5%

**Эксперименты:**
- Сравнение с/без кластеризации – кластеризация полезна
- Тюнинг глубины деревьев – глубина 6 оптимальна
- Добавление географических признаков – критически важно

**Вывод:** Для табличных данных градиентный бустинг (CatBoost) показывает лучшие результаты при меньших затратах на тюнинг.

**Ноутбуки:** `notebooks/03_baselines.ipynb`, `notebooks/04_advanced_models.ipynb`

## 5. Экспериментальный протокол и результаты

### 1. Экспериментальный протокол

**Разделение выборки:**
- Train/Test split в пропорции 80/20
- Random seed = 42 для воспроизводимости
- В ноутбуках — выборка 100 000 записей для ускорения экспериментов
- В `train.py` — полный очищенный датасет (641 362 записи)

**Подсчёт метрик:**
- Метрики считаются на test выборке (20% данных)
- Используются sklearn.metrics: mean_absolute_error, mean_squared_error, r2_score
- Дополнительная метрика MAPE рассчитывается вручную

### 2. Сравнение моделей по метрикам

**Примечание:** Метрики baseline-моделей получены в ноутбуках на выборке 100K. Финальная модель CatBoost обучена на полном датасете (641 362 записи, Москва и МО) в `train.py` с признаком `kmeans_cluster`.

| Модель / конфигурация | Описание | MAE (руб) | RMSE (руб) | R² | Комментарий |
|----------------------|----------|-----------|------------|-----|-------------|
| Linear Regression | Базовая модель (ноутбук, 100K, без кластера) | ~1 458 000 | ~1 798 000 | ~0.25 | Baseline |
| Decision Tree | max_depth=3 (ноутбук, 100K, без кластера) | ~1 312 000 | ~1 626 000 | ~0.38 | Нелинейная baseline |
| Bagging | 10 деревьев (ноутбук, 100K, с кластером) | ~839 000 | ~1 117 000 | ~0.71 | Ансамбль деревьев |
| CatBoost (ноутбук) | 100K, с кластером | ~398 000 | ~487 000 | ~0.94 | Промежуточный эксперимент |
| **CatBoost (prod)** | **train.py, 641K, с кластером** | **~422 000** | **~508 000** | **~0.94** | **Финальная модель** |

### 3. Выбор финальной модели

**Выбрана:** CatBoost Regressor

**Обоснование:**
1. **Лучшая точность:** MAE ≈ 422 000 руб на полном датасете — лучший результат среди протестированных моделей
2. **Скорость:** Время предсказания < 50ms на запрос
3. **Интерпретируемость:** CatBoost предоставляет важность признаков
4. **Устойчивость:** Менее чувствителен к выбросам по сравнению с деревьями
5. **Масштабируемость:** Обучен на 641 362 записях (Москва и МО)
6. **Простота тюнинга:** Хорошо работает с дефолтными параметрами

**Trade-off’ы:**
- CatBoost требует больше памяти для обучения, но это компенсируется качеством
- Время обучения больше, чем у линейной регрессии, но это разовая операция
- Для продакшена важнее скорость предсказания, а CatBoost предсказывает быстро

## 6. Архитектура решения и сервис

### 1. Архитектура пайплайна

**Схема пайплайна:**

```
data/all_v2.csv → src/data/prepare.py (очистка, IQR, фильтр Москва/МО)
    → data/cleaned_data.csv
    → src/models/train.py (K-Means + CatBoost)
    → artifacts/*.joblib
    → src/service/app.py (FastAPI)
```

**Компоненты:**
- `src/data/preprocessing.py` – очистка, IQR, фильтрация по региону
- `src/data/prepare.py` – подготовка `cleaned_data.csv`
- `src/models/train.py` – кластеризация, обучение CatBoost
- `src/service/app.py` – FastAPI сервис
- `artifacts/model.joblib`, `kmeans.joblib`, `feature_columns.joblib`

### 2. API и endpoints

**Реализованные endpoints:**

- `GET /health` – проверка работоспособности сервиса
  - Возвращает: `{"status": "ok", "model_loaded": true}`

- `POST /predict` – оценка стоимости недвижимости
  - Принимает: JSON с характеристиками объекта
  ```json
  {
    "area": 65.5,
    "kitchen_area": 10.0,
    "rooms": 2,
    "geo_lat": 55.7558,
    "geo_lon": 37.6173
  }
  ```
  - Возвращает:
  ```json
  {
    "predicted_price": 12500000,
    "investment_rating": "fair",
    "price_per_meter": 190839,
    "cluster": 2
  }
  ```

### 3. Технологический стек

**Библиотеки и фреймворки:**
- `FastAPI` – веб-фреймворк для API
- `CatBoost` – градиентный бустинг
- `scikit-learn` – ML инструменты (K-Means, метрики)
- `pandas`, `numpy` – обработка данных
- `joblib` – сериализация моделей
- `pydantic` – валидация данных
- `uvicorn` – ASGI сервер

**Docker:**
- Dockerfile для контейнеризации сервиса
- Команды:
  ```bash
  docker build -t real-estate-api .
  docker run -p 8000:8000 --env-file configs/.env real-estate-api
  ```

## 8. Ограничения и дальнейшая работа

**Текущие ограничения:**
- Используется только базовый набор признаков (без расстояний до метро, инфраструктуры)
- Модель обучена на данных за определённый период (может устаревать)
- Нет системы переобучения модели на новых данных
- Рекомендации основаны только на географическом proximity

**Дальнейшее развитие:**
- Добавление внешних признаков (расстояние до метро, парков, школ)
- Реализация RAG для анализа описаний объектов
- Добавление системы A/B тестирования для моделей
- Интеграция с картами для визуализации
- Добавление системы мониторинга качества модели в продакшене
